P1 · Anti-pattern banToast · Callout

Side-stripe purge on toasts and callouts

Toasts (app.css:1900-1907) and editor callouts (app.css:734-755) signal severity with a 3px colored left border. This is Impeccable's #1 absolute ban — it's the single tell most fluent reviewers spot first. The replacement keeps the same semantic clarity using a leading glyph chip in the role color plus a full subtle tinted border. Same information, no AI-slop stripe.

P1 · Architecturetokens.css · obsidian-scope.css

Scope token overrides under a body class

styles/tokens.css:79 redefines Obsidian's core CSS variables at :root — globally. Every Obsidian native chrome surface (tooltips, settings dialogs, the file rename modal, other plugins' modals) then reads WorkDesk values. The tooltip already broke and has a hardcoded patch at obsidian-scope.css:199-209; the next Obsidian update will break a different surface the same way. The fix is structural: apply WorkDesk tokens only where the plugin renders.

Before — :root override → native tooltip breaks
Hover area — tooltip above is what Obsidian renders when our tokens override --background-secondary-alt and --text-on-accent
After — scoped tokens → native chrome untouched
Hover area — Obsidian's native tooltip uses its own values, dark-on-light contrast intact, no patch needed
What changes in code: three structural edits.
  1. In styles/tokens.css: change :root, .theme-light { ... } → body.workdesk-os-active.theme-light, body.workdesk-os-active:not(.theme-dark) { ... }. Same for .theme-dark → body.workdesk-os-active.theme-dark. Only the --ws-* source tokens stay at :root.
  2. In src/main.ts: document.body.classList.add('workdesk-os-active') on plugin load, remove on unload. Also re-apply the class to every popout window via this.app.workspace.on('window-open', ...) so popouts get the design too.
  3. In styles/obsidian-scope.css: delete the tooltip patch (:199-209) and the .modal-container .modal visibility hack (:218-221). Both become unnecessary once the global override is gone.
/* tokens.css — diff */ :root, .theme-light { body.workdesk-os-active, body.workdesk-os-active.theme-light { --background-primary: /* warm-tinted, see finding 4 */; --text-normal: /* … */; /* … all Obsidian-native token reassignments … */ } .theme-dark { body.workdesk-os-active.theme-dark { /* dark theme overrides … */ } /* main.ts */ async onload() { /* … existing setup … */ document.body.classList.add('workdesk-os-active'); this.registerEvent(this.app.workspace.on('window-open', (win) => { win.doc.body.classList.add('workdesk-os-active'); })); } onunload() { document.body.classList.remove('workdesk-os-active'); }
P1 · TypographyP2 · Typography

Stop auto-capitalizing titles, replace synthetic italic

Capitalize: five sites (app.css:319, 362, 452, 511, .pane-hero h1) auto-capitalize titles. This mangles operator entity names — byrd-building, BenaliHQ, qbo-cli, WorkDesk OS — which carry real casing. Title casing is a writing decision, not a CSS decision.

Italic: six sites set font-style: italic against DM Sans and Manrope, neither of which has an italic woff2 in fonts/. Browsers synthesize the italic by skewing the upright. Particularly ugly on Manrope at display sizes.

P1 · Colortokens.css

Tint pure white and shadows toward warm amber

The warm-cream chrome reads as a unified system right up until a card or modal lands on #ffffff. Three pure-white assignments in tokens.css (:81 background-primary, :90 background-modifier-form-field, :99 text-on-accent) plus pure-gray shadows (rgba(20,20,20,…)) read as harsh-white islands instead of part of the same palette. Push the chroma 0.004 toward hue 75 (warm amber) and the surface is part of the room.

What changes in code: swap five color values + three shadow stacks in styles/tokens.css light-theme block. Dark theme already uses warm coffee values, so only light-mode adjustments are needed (one matching dark shadow update for consistency). Browser support: oklch() ships in all current browsers (Chrome 111+, Safari 15.4+, Firefox 113+); Obsidian's Electron runtime is Chromium 122+, no fallback needed. The plugin already uses color-mix(in oklch, …) in production at app.css:1125, 1202, 1625, 1626, 2028, so the format is already in use.
